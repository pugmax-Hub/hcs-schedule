# 歯科往診スケジュール作成ツール 🦷✨

このツールは、施設への歯科往診（訪問診療）のスケジュール作成、A4印刷用リスト（FAX用・院内用）の作成、および往診実績の管理を簡単に行うためのWebアプリです。

面倒な月ごとのデータ移動や、手書きでのスケジュール作成をすべて自動化します。

---

## 💡 このツールでできること

1. **自動で往診時間を計算**：
   医師や衛生士の訪問順を考慮し、治療に必要な時間を自動で割り振ってスケジュールを作成します。
2. **ワンタップ印刷**：
   「FAX送信表」「院内の往診予定表」「当月の往診回数表」を、すべてA4サイズにきれいに収まるよう自動調整して印刷できます。
3. **Excelデータベースの自動更新**：
   往診が終わった実績をアプリ上で記録し、ボタンひとつでExcelファイルに保存（上書き用のファイルをダウンロード）できます。手作業で日付を書き換える必要はありません。

---

## 🚀 使い方（かんたん3ステップ）

パソコンやスマートフォンのブラウザ（Google ChromeやSafariなど）で、公開されたホームページのURL（GitHub Pages）を開きます。

### 【ステップ 1】 Excelファイルを読み込む
1. 画面の真ん中にある点線の四角枠（ドラッグ＆ドロップエリア）をクリックします。
2. 管理している往診用のExcelファイル（例: `往診データベース.xlsx`）を選んで読み込ませます。
   - パソコンの場合は、ファイルを直接枠の中に引きずり込む（ドラッグ＆ドロップ）だけでも読み込めます。

### 【ステップ 2】 スケジュールを組む
1. **「対象施設」**を選びます。（Excel内のシート名が自動で選択肢になります）
2. **「往診日」**や同行する**「医師」「衛生士」**の名前、開始時間を選びます。
3. 左側の「患者一覧」から、今日往診する患者さんのチェックボックスにチェックを入れます。
4. 右側の「本日の往診スケジュール」に患者さんが追加されます。
   - 患者さんカードの左端（三本線マーク）を上下に引っ張る（ドラッグ）ことで、**往診する順番を自由に入れ替える**ことができます。
   - 各患者さんの「実施/不実施」や「往診タイプ（医師＋衛生士／衛生士のみ）」もここで切り替えられます。

### 【ステップ 3】 印刷と保存をする
- **印刷する**：
  上部のメニューから「② 往診リスト印刷」または「③ FAX送信表印刷」を選び、**「このリストを印刷」**ボタンを押すと、お使いのプリンターからA4用紙で印刷できます。
- **データを保存する**：
  1. すべての往診が終わったら、上部メニューの「④ 往診回数表印刷」を開きます。
  2. 緑色の **「結果をデータベースへ保存 (Excel出力)」** ボタンを押します。
  3. 新しいExcelファイルがダウンロードされますので、**元のExcelファイルに上書き保存（差し替え）**してください。これで、誰が何回往診を受けたかのデータが自動で最新になります。

---

## 📊 Excelファイル（データベース）のつくり方

Excelファイルを用意するときは、以下のルールで作成してください。

- **シート名**：施設名（例: `鴻巣フラワーパレス`、`川里苑` など）にします。
- **1行目の見出し（列の名前）**：以下の名前を正確に入力してください。
  ```
  部屋番号 | カルテNo | 名前 | 義歯 | 1回目診察日 | 2回目診察日 | 注意事項 | 備考
  ```
  ※診察日の列は、最大4つ（「診察日1」〜「診察日4」など）まで自動で増やすことができます。

## 🔒 セキュリティと個人情報について

> [!IMPORTANT]
> **個人情報の入ったExcelファイルは、絶対にGitHubなどのインターネット上にアップロード（公開）しないでください。**
> 
> **なぜ安全なのですか？**
> このアプリは、Excelファイルを読み込ませても、そのデータをインターネット上のサーバーに送信していません。データ処理はすべて「あなたのパソコンやスマホの中（ブラウザ）」だけで行われているため、外部に患者さんの情報が漏れる心配はありません。
> 
> **注意点**
> GitHubにこのプログラム（`index.html`）をアップロードする際は、患者さんの名前やカルテNoが入った本番用のExcelファイル（`往診データベース.xlsx`）はアップロード対象に入れないように十分注意してください。（見本用の dummy データが入ったExcelファイルのみにしておくことを推奨します）。
